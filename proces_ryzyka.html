def N_distr_gen(n):
    """Generate random variable from discreet distribution.

    Args:
        n [int]: actual count of OC clients.

    Returns:
        [int]: random variable.
    """
    u = np.random.uniform(0, 1)
    if u <= 0.5:
        return n
    elif 0.5 < u <= 0.8:
        return np.floor(0.9*n)
    else:
        return np.floor(n/2)

Generowanie liczby kierowców rezygnujących z OC na początku miesiąca.¶

Warto popytu na polisy OC ($m_{OC}$) powinna by mniejsza lub rwna wartoci ceny polisy ($c_{OC}$), eby drugi parametr rozkadu dwumianowego (parametr $p$ - prawdopodobiestwo) by mniejszy lub rwny 1. W przeciwnym wypadku rozkad nie bdzie dziaa (poniewa prawdopodobiestwo nie moe przekracza 1).
Jeeli jednak parametr $p$ przekroczy warto 1, wtedy za $p$ podstawiamy 0.9 (w celu uniknicia bdw).

def count_quit_OC_gen(n, c_OC, m_OC):
      """Generate count of clients resign OC.

      Args:
          n [int]: actual count of OC clients,
          c_OC [int, float]: OC policy price,
          m_OC [int, float]: demand for OC policy.

      Returns:
          [int]: count of clients resign OC.
      """
      if n == 0:
        return 0
      p = (c_OC/m_OC)**2.5
      if p > 1:
          p = 0.9
      return np.random.binomial(N_distr_gen(n), p)

Rezygnacja z AC.¶

Generowanie liczby kierowców rezygnujących z AC na początku miesiąca.¶

Zmienna $Z$ zwraca liczby z rozkadu $N(0, 1)$ (z duym prawdopodobiestwem s to wartoci z przedziau od -3 do 3), dlatego te zmienna $|Z|$ ma wartoci z przedziau od 0 do 3. eby parametr $p = |Z|\frac{c_{AC}}{m_{AC}}$ w rozkadzie dwumianowym by mniejszy lub rwny 1, warto $m_{AC}$ musi by co najmniej 3 razy wiksza ni warto $c_{AC}$ (inaczej rozkad nie bdzie dziaa).
Jeeli jednak parametr $p$ przekroczy warto 1, wtedy (analogicznie do przypadku generowania liczby rezygnacji z OC) za $p$ podstawiamy 0.9.

def count_quit_AC_gen(m, c_AC, m_AC):
    """Generate count of clients resign AC.

    Args:
        m [int]: actual count of AC clients,
        c_AC [int, float]: price of AC policy,
        m_AC [int, float]: demand for AC policy.

    Returns:
        [int]: count of clients resign AC.
    """
    Z = np.random.normal(0, 1)
    p = np.abs(Z)*(c_AC/m_AC)
    if p > 1:
        p = 0.9
    return np.random.binomial(m, p)

Wnioski:¶

Wraz ze zwikszaniem si popytu na polisy, prawdopodobiestwo rezygnacji z ubezpiecze zmniejsza si, a co za tym idzie mniej klientw odchodzi z naszej firmy.
Im wiksze ceny polis, tym wicej klientw rezygnuje z kupowania polis w naszej firmie.

Generowanie liczby nowo sprzedanych polis OC na początku każdego miesiąca.¶

Nakady na marketing ($\mu$) zwracaj nam si w postaci nowych klientw.
eby marketing by dla nas opacalny, generujc parametr $\lambda = 2\log(1 + 1000\mu\sqrt{\frac{M}{c_{OC}} + \frac{s_{OC}}{c_{OC}}})$ w rozkadzie Poissona mnoymy $\mu$ razy 1000.
W reklam inwestujemy sta kwot rwn 1000 z - jest to dla nas najbardziej opacalne - przy wikszych nakadach liczba naszych klientw i tak nie bdzie znaczco rosn.
Parametr $s_{OC}$ (staa zwizana z popytem) ma u nas niezmienn warto rwn 100. Rwnie parametr $n_0$ jest stay i wynosi 100.
Aktualna liczba klientw (w tym take parametr $n_0$) nie ma duego wpywu na ilo nowo sprzedanych polis.

def M_distr_gen(n, n_0):
    """Generate random variable from discreet distribution.

    Args:
        n [int]: actual count of OC clients,
        n_0 [int, float]: fixed parameter.

    Returns:
        [int]: random variable.
    """
    u = np.random.uniform(0, 1)
    if u <= 0.1:
        return n/2
    elif 0.1 < u <= 0.2:
        return n/3
    elif 0.2 < u <= 0.6:
        return n/5
    else:
        return n_0

def count_sale_OC_gen(n, n0, s_OC, c_OC, mu):
    """Generate count of sale OC policy.

    Args:
        n [int]: actual count of OC clients,
        n_0 [int, float]: fixed parameter,
        s_OC [int, float]: constant related to the demand,
        c_OC [int, float]: OC policy price,
        mu [int, float]: marketing price.

    Returns:
        [int]: count of sale OC.
    """
    M = M_distr_gen(n, n0)
    advert = 1000*mu
    lamb = 2*np.log(1 + advert*np.sqrt(M/c_OC) + s_OC/c_OC)
    return np.random.poisson(lamb)

Generowanie liczby nowo sprzedanych polis AC na początku każdego miesiąca.¶

Aktualna liczba naszych klientw OC ma bardzo duy wpyw na to, ile nowych polis AC sprzedamy. Liczba nowych klientw AC jest odwrotnie proporcjonalna do iloczynu iloci klientw OC oraz ceny polisy AC. Co oznacza, e im wicej klientw OC kupuje polisy w naszej firmie, tym mniej klientw AC zacznie do nas przychodzi.
Ponadto gdy nie mamy adnych klientw AC na pocztku, klienci ci nie zaczn si pojawia.
W celu uniknicia bdnego dziaania programu, gdy parametr $p$ w rozkadzie dwumianowym przekroczy warto 1, wtedy za $p$ podstawiamy 0.9.

def count_sale_AC_gen(n, m, c_AC):

Odszkodowania OC.¶

Generator czasu pojawiania się szkód dla wszystkich klientów, którzy kupili OC.¶

Generator zwraca czasy szkd w cigu jednego miesica.
Funkcja intensywnoci ($\lambda (t)$) zaley od konkretnego okresu w roku oraz iloci sprzedanych polis OC i jest opisana wzorem: $$\lambda(t) = \sum_{i=0}^{n(t)}W_i\cdot \lambda_0(t),$$ gdzie:

$n(t)$ - ilo sprzedanych w danym miesicu polis (wliczamy staych i nowych klientw),
$W_i \sim Exp(\lambda_{wi})$ - zmienna opisujca jak czsto dany kierowca miewa stuczki,
$\lambda_0(t)$ - intensywno wypadkw w zalenoci od okresu w roku.

Dobieranie parametru intensywności $\lambda_0$ w zależności od danego miesiąca.¶

Procentowy rozkad wypadkw dla danego miesica w roku 2020:

Na podstawie danych z tabelki dobierzemy intesywno $\lambda _0$.
W naszym przypadku rednia intensywno wypadkw (warto referencyjna) wynosi 1, dlatego procentowe dane z tabelki skalujemy wedug tej redniej. rednia z tabelki wynosi okoo $8.3\%$. Przykadowo przeskalowana intensywo dla stycznia wynosi $\frac{8.5\%}{8.3\%} \approx 1.03$.

def lamb(count_of_sale, lamb_wi, month):
    """Generate Poisson intensity function depend on month.
    Args:
        count_of_sale [int]: count of sale OC policy,
        lamb_wi [int, float]: parametr in exponential distribution,
        month [int]: number from 1 to 12.

    Returns:
        [float]: intensity.
    """
    return sum([np.random.exponential(1/lamb_wi)*lamb0(month) for __ in range(count_of_sale)])

def accident_OC_time_gen(count_of_sale, lamb_wi, month, T=30):
    """Generate accidents times for clients with OC policy.

    Args:
        count_of_sale [int]: count of sale OC policy,
        lamb_wi [int, float]: parametr in exponential distribution,
        month [int]: number from 1 to 12,
        T (int, optional): time range in days. Defaults to 30.

    Returns:
        [list]: list with accidents times.
    """
    S = []
    t = 0
    mybool = True
    l = lamb(count_of_sale, lamb_wi, month)
    if l > 0:
        while mybool:
            U = np.random.uniform(0, 1)
            t = t - (1/l)*np.log(U)
            if t > T:
                mybool = False
            else:
                S.append(t)
    else:
        S = []
    return S

def accident_AC_time_gen(T=30):
    """Generate accidents times for clients with AC policy.

    Args:
        T (int, optional): time range in days. Defaults to 30.

    Returns:
        [list]: list eith accidents times.
    """
    S = []
    t = 0
    mybool = True
    while mybool:
        V = np.random.rayleigh(100*np.sqrt(np.pi/2))
        tv = t + 20
        u = np.random.uniform(tv, tv+V)
        t = t + u
        if t > T:
            mybool = False
        else:
            S.append(t)
    return S

Czas oczekiwania na sygna jest zmienn z rozkadu $U(t_v, t_v + V)$.
W naszym przypadku $t_v = t + 20$ jest zmienn zalen od czasu poprzedniego skoku. Poniewa generator zwraca czasy wypadkw dla jednego klienta w cigu miesica, nie powinno by ich duo, a najczciej nie bdzie ich wcale.

Funkcja licząca wartość przy której automatycznie bankrutujemy.¶

Jest ona zdefiniowana wzorem: $$bankructwo = -2k_0 - nc_{AC},$$ gdzie:
$k_0$ - kapita pocztkowy,
$n$ - ilo aktualnych klientw, ktrzy kupili OC w naszej firmie,
$c_{AC}$ - cena polisy AC.

def bankruptcy_value(n, k0, c_AC):
    """Count bankruptcy value.

    Args:
        n [int]: actual count of OC clients,
        k0 [int, flaot]: start-up capital,
        c_AC [int, float]: AC price.

    Returns:
        [flaot]: bankruptcy value.
    """
    return -2*k0 - n*c_AC

Wartość podatku, który płacimy raz w roku.¶

Funkcj liczc warto podatku definiujemy nastpujco: $$podatek = 0.3\cdot z + 0.05\cdot p + 0.04 \cdot k,$$ gdzie:
$k$ - kapita - aktualny kapita naszej firmy,
$p$ - przychd - czna warto sprzedanych polis OC i AC w cigu roku,
$z$ - zysk - rnica przychodu i kosztw ich uzyskania (gdzie do kosztw wliczamy: nakady na marketing ($\mu$), wypacane odszkodownia OC i AC oraz jeli zacignlimy kredyt doliczmy oprocentownie w wysokoci $r$).
Odsetki z caegu kredytu wynosz: $20 \cdot \frac{1}{12} kredytu - kredyt = \frac{2}{3} kredytu$, co miesic do kosztw uzyskania przychodu doliczamy $\frac{1}{20}$ tych odetek: $$r = \frac{1}{20}\cdot\frac{2}{3} kredytu = \frac{1}{30} kredytu.$$

def pay_tax(year_income, year_loss, capital):
    """Count value of tax.

    Args:
        year_income [int, float]: income in year,
        year_loss [int, float]: cost of getting income,
        capital [int, float]: actual capital.

    Returns:
        [flaot]: value of tax to pay.
    """
    profit = year_income - year_loss
    profit_tax  = 0
    capital_tax = 0
    income_tax = 0.05*year_income
    if profit > 0:
        profit_tax = 0.3*profit
    if capital > 0:
        capital_tax = 0.04*capital
    return  profit_tax + income_tax + capital_tax

Symulacja procesu ryzyka¶

ALGORYTM:¶

Pocztek i koniec symulacji podajemy jako numer miesica. Lata podajemy przeliczone na miesice.

Dla kadego miesica z podanego przedziau czasu:

  1. Sprawdzamy czy zbankrutowalimy, jeli nie, wykonujemy nastepujce kroki:

    1.1. Sprawdzamy czy spacamy kredyt, jeeli tak, to obliczamy rat kredytu i doliczamy oprocentowanie do kosztw uzyskania przychodu.

    1.2. Na podstwie strategii na marketing, sprawdzamy czy inwestujemy w reklam w danym miesicu.

    1.3. Obliczamy aktualn liczb klientw (dodaje nowych i odejmujemy, tych ktrzy zrezygnowali) oraz sprawdzamy czy ich liczba nie jest ujemna (jeeli tak jest, to podstawiamy 0 za liczb klientw).

    1.4. Rejestrujemy miesiczny oraz roczny przychd.

    1.5. Zapisujemy kapita (dla danego miesica) powikszony o przychody. Jeeli kapita w tym momencie jest dodatni, koczymy odliczanie czasu ruiny (licznik zerujemy).

    1.6. Obliczamy wielkoci odszkodowa, ktre pacimy w danym miesicu.

    1.7. Obliczamy aktualny kapita, od ktrego bdziemy paci podatek.

    1.8. Jeeli jest kwiecie pacimy podatek i odejmujemy go od kapitau.

    1.9. Zapisujemy aktualny kapita.

    1.10. Sprawdzamy czy kapita jest ujemny, jeeli jest, to zwikszamy czas ruiny i jeeli nie mamy adnych zobowiza, bierzemy kredyt.

    1.11. Sprawdzamy, czy zarabiamy milion.

    1.12. Jeeli nasz kapita osiga $10^6$ (i $\textit{million_check=True}$) lub skoczy si czas symulacjizwracamy tablice czasu i kapitau.

  2. W przypadku bankructwa zatrzymujemy proces ryzyka i zwracamy tablice wartoci do momentu bankructwa.

def risk_process(start, stop, k0, n, m, c_AC, c_OC, m_OC=200, m_AC=240, s_OC=100, lamb_wi=500, M_AC=10, million_check=False):
    """Generate risk process.

    Args:
        start [int]: month when simulation starts,
        stop [int]: month when simulation stops,
        k0 [float, int]: start-up capital,
        n [int]: initial number of OC clients,
        m [int]: initial number of AC clients,
        c_AC [float, int]: AC price,
        c_OC [float, int]: OC, price,
        m_OC (int, optional): demand for OC policy. Defaults to 200.
        m_AC (int, optional): demand for AC policy. Defaults to 240.
        s_OC (int, optional): demand parameter. Defaults to 100.
        lamb_wi (int, optional): exponential distribution parameter. Defaults to 500.
        M_AC (int, optional): multiply of AC price parameter. Defaults to 10.
        million_check (bool, optional): boolean to check if capital has reached a million. Defaults to False.

    Returns:
        [tuple with 2 lists]: list with time data, list with capital data.
    """
    R_range = [k0]
    time_range = [0]
    bankruptcy = bankruptcy_value(n, k0, c_AC)
    k = 0
    t = 0
    R = k0
    n_var = n
    m_var = m
    credit_time = False
    credit_start_counter = 26
    credit_rate = 0
    ruin_counter = 0
    tax = 0
    year_income = 0
    year_loss = 0
    mu = 1000
    millionaire = 0
    for t in range(start, stop+1):
        if R > bankruptcy and ruin_counter < 24:
            if credit_time:
                #jeeli zacignlimy kredyt zaczynamy odliczanie p roku
                credit_start_counter -= 1
            #po upywie p roku zaczynamy spaca kredyt
            #sprawdzamy czy ju spacilimy kredyt
            if 0 < credit_start_counter <= 20:  
                credit_rate = (1/12)*credit
                credit_start_counter -= 1
                #jeeli spacamy kredyt, to do kosztw uzysaknia przychodu doliczmy oprocentowanie kredytu
                year_loss += (1/30)*credit
            #po spaceniu kredytu liczniki zerujemy 
            elif credit_start_counter <= 0 :
                credit_time = False
                credit_start_counter = 26
                credit_rate = 0

            #sprawdzamy czy inwestujemy w marketing
            if mu > 5*R:
                mu = 0
            else:
                mu = 100
            
            clients_quiting_OC = count_quit_OC_gen(n_var, c_OC, m_OC) #ilo klientw rezygnujca z OC
            clients_quiting_AC = count_quit_AC_gen(m_var, c_AC, m_AC) #ilo klientw rezygnujca z AC
            sale_OC = count_sale_OC_gen(n_var, n0, s_OC, c_OC, mu)
            sale_AC = count_sale_AC_gen(n_var, m_var, c_AC)
            n_var = n_var + sale_OC - clients_quiting_OC
            m_var = m_var + sale_AC - clients_quiting_AC

            #sprawdzamy czy liczba klientw nie jest mniejsza od 0 (liczba klientw nie moe by ujemna)
            if n_var < 0:
                n_var = 0
            if m_var < 0:
                m_var = 0

            income = monthly_income(n_var, c_OC, m_var, c_AC)
            #rejestrujemy roczne przychody
            year_income += income
            #zapisujemy kapita powikszony o przychody
            R_plus = R + income
            R_range.append(R_plus)
            time_range.append(t)
            #sprawdzamy czy jestemy na plusie
            if R_plus > 0:
                ruin_counter = 0

            #obliczymy wielko odszkodowa dla wszystkich aktualnych klientw
            OC_compensation = monthly_OC_compensation_value(n_var, lamb_wi, t)
            AC_compensation = monthly_AC_compensation_value(m_var, M_AC, c_AC)
            
            year_loss += OC_compensation + AC_compensation + mu

            #na koniec kwietnia pacimy podatek (R to nasz aktualny kapita)
            R = R + income - OC_compensation - AC_compensation - credit_rate - mu
            if t%12 == 4:
                tax = pay_tax(year_income, year_loss, R)
                #po zapaceniu podatku zerujemy licznik
                year_income = 0
                year_loss = 0
            else:
                tax = 0         
            
            #zapisujemy kapita pomniejszony o odszkodowania, podatek, rat kredytu, marketing
            R = R - tax
            R_range.append(R)
            time_range.append(t)
            #na koniec miesica sprawdzamy, czy kapita jest ujemny
            if R <= 0:
                ruin_counter += 1
            #sprawdzamy czy bierzemy kredyt
            if R <= 0 and credit_time == False:
                credit = credit_gen(R)
                R += credit
                credit_time = True
            bankrupty = bankruptcy_value(n_var, k0, c_AC)
            if million_check and R >= 10**6:
                millionaire += 1
                return time_range, R_range, millionaire     
        else:
          #po bankructwie zwracamy od razu tablice
          return time_range, R_range, millionaire
    return time_range, R_range, millionaire

Analizy przeprowadzone w celu ustalenia strategii i tego jak nasza firma będzie rozwijać się na przestrzeni czasu.¶

Strategia na marketing, zaciąganie kredytu, wysokości cen OC i AC.¶

MARKETING.¶

Jak ju wczeniej ustatlilismy nie opaca nam si inwestowa w marketing wicej ni 1000 z. Strategia polega na tym, e jeeli nasz kapita jest 5 razy wikszy ni nakad na marketing (wynosi wicej ni 5000 z), wtedy inwestujemy w marketing, w przeciwnym wypadku rezygnujemy z niego w danym miesicu, eby nie ryzykowa utraty pienidzy.

KREDYT.¶

Jeeli nasz kapita spadnie poniej 0, wtedy zacigamy kredyt. Wysoko kredytu wynosi: $|R| + 10000$, gdzie $R$ to aktualny kapita. Warto ta jest dobrana w taki sposb, ebymy po wziciu kredytu mieli 10000 z kapitau i mogli ustabilizowa nasze dochody.
Kredyt zacigamy na koniec miesica, jeeli mamy kwecie, to po zapaceniu podatkw (opaca nam si to najbardziej, poniewa przed zacigniciem kredytu mamy ujemny kapita, z ktrego nie musimy paci podatkw).

def capital_simulation(start, stop, k0, n, m, c_OC, c_AC):
    """Count capital and bankruptcy value

        [tuple with 4 float values]: maximum capital value, minimum capital value,

    return round(min(capital_tab), 2), round(max(capital_tab), 2), round(sum(capital_tab)/1000, 2), bankruptcy/10

def ruin_prob_simulation(k0, list_nm, c_OC, c_AC, start=1, stop=60):

    Args:
        k0 [float, int]: start-up capital,

        [list]: list with bankruptcy values.
    """
    bankruptcy_tab = []
    for value in list_nm:
        bankruptcy = 0
        for __ in range(1000):
